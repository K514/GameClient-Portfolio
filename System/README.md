## System

게임 실행 흐름(부팅 → 진입 → 업데이트)을 관리하는 시스템입니다.  
Boot, Entry, Loop 세 단계로 나누어 전체 게임 생명주기를 제어합니다.

---

### 개요

게임 실행 시 필요한 초기화, 씬 진입, 런타임 업데이트를  
단계별로 분리하여 관리하도록 설계했습니다.

---

## 구조

### ■ Boot
📂 `System/Boot/`

게임 실행 시 가장 먼저 수행되는 초기화 단계입니다.

---

#### 역할

- 핵심 싱글톤 초기화
- 리소스 로더 및 테이블 로더 초기화
- 시스템 메타 데이터 테이블 초기화

---

#### 초기화 특징

- 싱글톤을 **단계별(Level)로 나누어 초기화**
- 의존성에 따라 초기화 순서를 보장

📌 예시
- FirstPass → 리소스/로더
- SecondPass → 시스템 테이블
- ThirdPass → 추가 데이터

👉 초기화 순서를 명확하게 관리하여 안정성 확보

---

#### 그룹 기반 로딩

실행 모드에 따라 로드할 시스템을 분리합니다.

📌 예시
- Debug Mode
  - 최소 시스템만 로드 (카메라, 오디오 제외)
- Game Mode
  - 전체 시스템 로드

👉 환경에 따라 유연한 초기화 가능

---

## ■ Entry
📂 `System/Entry/`

Boot 이후, 최초로 진입할 씬을 결정하는 단계입니다.

---

#### 역할

- 초기 진입 씬 결정
- 실행 모드에 따른 분기 처리

📌 예시
- 일반 실행 → 타이틀 씬
- 디버그 모드 → 디버그 씬

---

## ■ Loop
📂 `System/Loop/`

게임 실행 중 반복되는 업데이트를 관리합니다.

---

#### 역할

- 각 싱글톤 매니저의 Update 호출
- 게임 전체 업데이트 흐름 제어

---

#### 설계 특징

Unity의 개별 MonoBehaviour Update에 의존하지 않고,  
중앙에서 업데이트를 관리하도록 구성했습니다.

---

#### 활용

- TimeScale 제어
- 특정 시스템 일시 정지

📌 예시
- 입력 매니저 비활성화
- 게임 로직 일시 정지

---

## 설계 특징

### 1. 실행 흐름 분리
- Boot / Entry / Loop 단계 분리
- 게임 생명주기 명확화

---

### 2. 초기화 순서 제어
- 싱글톤 의존성 기반 초기화
- 단계별 초기화 구조

---

### 3. 그룹 기반 시스템 로딩
- 실행 환경에 따라 시스템 선택적 로딩

---

### 4. 중앙 업데이트 구조
- Update 흐름을 한 곳에서 제어
- 시스템 간 동기화 용이

---

## 확장성

- 새로운 시스템 추가 시 기존 구조 변경 최소화
- 실행 모드 추가 대응 가능
- 초기화 단계 확장 가능

---

## 확인 포인트

- Boot 단계의 싱글톤 초기화 순서
- Entry 단계의 씬 분기 처리
- Loop 단계의 중앙 업데이트 구조
