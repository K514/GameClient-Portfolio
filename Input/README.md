## Input System

다양한 입력 장치(키보드, 마우스, 터치)를 통합하고,  
커맨드 입력 기반 스킬 발동을 지원하는 입력 시스템입니다.

---

### 개요

던전 앤 파이터의 커맨드 입력 시스템을 참고하여,  
단순 입력 처리뿐 아니라 **입력 조합 기반 스킬 발동 구조**를 구현했습니다.

---

### 입력 처리 구조

입력 흐름은 다음과 같습니다.

1. 입력 발생 (키보드 / 마우스 / 터치)
2. 입력 → 내부 KeyCode로 매핑
3. 매핑된 입력을 이벤트 리시버들에게 전파

---

### 키 매핑 시스템

입력은 내부 KeyCode로 변환되어 처리됩니다.

- 키 변경(리맵핑)은 KeyCode 간 교체로 구현
- 입력 장치와 관계없이 동일한 방식으로 처리

---

### 키 타입 분류

입력은 3가지 타입으로 구분됩니다.

#### ■ ArrowKey (방향 입력)
- 방향키 및 조이스틱 입력 통합
- 출력 데이터:
  - 방향 타입 (8방향 enum)
  - 방향 벡터 (정규화된 uv)

📌 예시
- 키보드: ↑ + → → RightUp + (1,1).normalized  
- 조이스틱: 1시 방향 → RightUp + 실제 uv 값  

👉 플랫폼에 관계없이 동일한 방향 입력 처리 가능

---

#### ■ TriggerKey (행동 입력)
- 스킬, 공격 등 실제 게임 플레이 입력
- 예: Z, X, C, V, A, S, D, F

---

#### ■ FunctionKey (시스템 입력)
- UI 및 시스템 제어용 입력
- 예: ESC

👉 게임 일시정지 시:
- TriggerKey 차단
- FunctionKey만 허용

---

### 커맨드 입력 시스템

입력 이벤트 리시버 중 **Command Event Receiver**가 존재하며,  
다음과 같은 방식으로 동작합니다.

---

#### 처리 방식

1. ArrowKey 입력을 최대 4개까지 버퍼에 저장
2. TriggerKey 입력 발생 시:
3. 방향키 + 트리거키 조합 → 시퀀스 키 생성
4. 커맨드 이벤트로 다시 전파

---

#### 결과

- 커맨드 입력 기반 스킬 발동 가능
- 실제 던전 앤 파이터와 유사한 입력 시스템 구현

---

### 설계 특징

#### 1. 입력 장치 독립성
- 키보드 / 터치 / 조이스틱 입력을 하나의 데이터로 통합

---

#### 2. 이벤트 기반 구조
- 입력을 직접 처리하지 않고 이벤트로 전파
- 시스템 간 결합도 감소

---

#### 3. 입력 타입 분리
- 게임 입력 / 시스템 입력 구분
- 상태에 따른 입력 제어 가능

---

#### 4. 커맨드 기반 확장성
- 입력 조합만으로 다양한 스킬 구현 가능
- 새로운 커맨드 추가 용이

---

### 확인 포인트

- ArrowKey → 방향 통합 처리 구조
- KeyCode 매핑 및 리맵 시스템
- Command 입력 버퍼 및 시퀀스 생성 방식
- 이벤트 기반 입력 전파 구조
