# GameClient-Portfolio

Unity 기반 게임 클라이언트 시스템을 설계 및 구현한 포트폴리오입니다.  

단순 기능 구현이 아닌,  
**확장성, 재사용성, 시스템 분리 구조를 고려한 아키텍처 설계**를 중심으로 개발했습니다.

- GitHub Pages: https://K514.github.io/GameClient-Portfolio
- 웹 포트폴리오 소스: `docs/`

## 📌 프로젝트 개요

이 프로젝트는 게임 클라이언트 전반을 구성하는 핵심 시스템을  
직접 설계하고 구현하는 것을 목표로 개발되었습니다.

- 게임 실행 흐름 (Boot / Entry / Loop)
- 데이터 중심 설계 (Table System)
- 모듈 기반 엔터티 시스템
- 입력 → 행동 → 스킬 커맨드 구조
- UI / Scene / Camera / Pool 통합 관리

---

## 🧩 핵심 설계 특징

### ■ 1. 모듈 기반 엔터티 시스템
- 기능을 클래스 상속이 아닌 **모듈 조합 방식**으로 구성
- AI, Action, Stat 등을 동적으로 교체 가능

📌 결과  
- 몬스터 → NPC 전환  
- 패턴 변경 (보스 2페이즈)  
- 탄막에 AI 부여 등 확장 가능

---

### ■ 2. 입력 통합 + 커맨드 시스템
- 키보드 / 터치 / 조이스틱 입력을 하나의 구조로 통합
- 방향 입력을 ArrowKey로 추상화
- 커맨드 입력 버퍼를 통해 스킬 발동 구현

📌 결과  
- 플랫폼에 관계없는 입력 처리  
- 던전 앤 파이터 스타일 커맨드 시스템 구현

---

### ■ 3. 데이터 중심 테이블 시스템
- XML 기반 데이터 → 객체 자동 매핑
- Dictionary / List / Tuple 파싱 지원
- 바이트 직렬화를 통한 로딩 최적화

📌 결과  
- 데이터만으로 게임 로직 확장 가능  
- XML 파싱 제거로 성능 개선

---

### ■ 4. 게임 실행 흐름 제어 (System)
- Boot / Entry / Loop 구조로 생명주기 분리
- 싱글톤 의존성 기반 초기화 순서 제어
- 중앙 Update 관리 구조

📌 결과  
- 안정적인 초기화  
- 시스템 간 충돌 최소화

---

### ■ 5. 리소스 및 수명 관리 시스템
- 오브젝트 풀링 + 수명 타입(Scene 등) 관리
- 씬 전환 시 자동 회수

📌 결과  
- 메모리 관리 안정성  
- 런타임 성능 최적화

---

### ■ 6. 씬 시스템 + 환경 분리 구조
- Enum 기반 씬 접근
- SceneEnvironment로 씬 로직 분리
- 씬별 프리로드 시스템

📌 결과  
- 동일 씬을 다양한 역할로 재사용 가능  
- 씬 로직 독립 관리

---

### ■ 7. UI 컨테이너 + 이벤트 구조
- UI 계층 구조(Container)
- 입력 및 이벤트 전파 구조

📌 결과  
- UI 상태 동기화  
- 입력 시스템과 자연스럽게 연동

---

### ■ 8. 싱글톤 아키텍처 확장
- 클래스 / MonoBehaviour / ScriptableObject / Editor 분리
- 동기 / 비동기 초기화 지원
- 의존성 기반 초기화

📌 결과  
- Unity 환경에 맞는 안정적인 시스템 구성

---

## 📂 주요 폴더

- `GameEntity/` → 모듈 기반 엔터티 시스템
- `Table/` → 데이터 관리 및 파싱 시스템
- `Input/` → 입력 및 커맨드 처리
- `UI/` → UI 및 이벤트 구조
- `Scene/` → 씬 관리 및 환경 시스템
- `System/` → 게임 실행 흐름 관리
- `Pool/` → 오브젝트 풀링
- `DataStorage/` → 데이터 저장 시스템
- `CameraManager/` → 카메라 제어 시스템
- `Singleton/` → 싱글톤 및 초기화 구조
- `docs/` → GitHub Pages 기반 웹 포트폴리오
