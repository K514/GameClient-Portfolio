## Data Storage

게임 설정 및 계정 정보 등을 저장하고 관리하는 시스템입니다.

---

### 개요

게임 데이터를 XML 형태로 저장하며,  
저장 위치를 상황에 따라 유연하게 변경할 수 있도록 설계되었습니다.

---

### 주요 역할

- 게임 설정 데이터 저장
- 계정 및 사용자 데이터 관리
- 데이터 읽기 / 쓰기 인터페이스 제공

---

### 저장 방식

데이터는 XML 형식으로 직렬화되어 저장됩니다.

또한, 프로젝트 환경에 따라 저장 위치를 분기할 수 있습니다.

지원 저장소:

- PlayerPrefs (로컬 저장)
- Steam (플랫폼 연동 저장)
- Backend (서버 저장)

---

### 설계 특징

#### 1. 저장소 분리 구조
데이터의 저장 위치와 로직을 분리하여,  
저장 방식이 변경되더라도 기존 코드 수정 없이 대응 가능하도록 설계했습니다.

---

#### 2. 환경 기반 분기 처리
프로젝트 상태 또는 실행 환경에 따라  
데이터 저장 위치를 동적으로 선택할 수 있습니다.

---

#### 3. 공통 인터페이스 제공
데이터 읽기/쓰기 로직을 하나의 인터페이스로 통합하여  
상위 로직에서 저장 방식에 의존하지 않도록 구성했습니다.

---

### 확장성

- 새로운 저장소(예: 클라우드, DB 등) 추가 시 구조 변경 최소화
- 플랫폼별 저장 방식 대응 가능
- 멀티 환경(로컬/서버) 지원 가능

---

### 확인 포인트

- 저장소 분리 구조 (Storage abstraction)
- 환경에 따른 저장 분기 처리
- 데이터 접근 로직의 일관성 유지
