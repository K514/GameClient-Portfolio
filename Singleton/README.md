## Singleton System

게임 전반에서 사용하는 싱글톤 객체를  
유형과 초기화 방식에 따라 분리하여 관리하는 시스템입니다.

---

### 개요

Unity 환경에서 사용되는 객체들은  
각각 다른 생명주기(Lifecycle)를 가지기 때문에,  
단일한 싱글톤 구조로는 안정적인 관리가 어렵습니다.

이를 해결하기 위해,  
객체 타입별로 싱글톤을 분리하여 설계했습니다.

---

### 싱글톤 타입 분류

객체 유형에 따라 4가지 싱글톤으로 나뉩니다.

#### ■ 일반 클래스 싱글톤
- 순수 C# 클래스 대상
- 게임 로직 및 데이터 관리용

---

#### ■ Unity 컴포넌트 싱글톤
- MonoBehaviour 기반
- 씬에 존재하는 오브젝트 관리

---

#### ■ ScriptableObject 싱글톤
- 데이터 중심 객체 관리
- 에셋 기반 데이터 공유

---

#### ■ Editor 전용 싱글톤
- Unity Editor 환경에서만 사용
- 에디터 툴 및 기능 관리

---

### 설계 의도

각 객체 타입은 서로 다른 생성 방식과 생명주기를 가지기 때문에,  
이를 하나의 싱글톤으로 통합하지 않고 **유형별로 분리**했습니다.

---

### 초기화 방식 (동기 / 비동기)

각 싱글톤은 두 가지 초기화 방식 중 선택할 수 있습니다.

#### ■ 동기 싱글톤
- 즉시 초기화
- 간단한 시스템에 적합

---

#### ■ 비동기 싱글톤
- 비동기 로딩 기반 초기화
- 대용량 데이터 또는 리소스 로딩에 적합

---

### 의존성 초기화 시스템

싱글톤은 초기화 시,  
자신이 의존하는 다른 싱글톤을 먼저 초기화하도록 설계되어 있습니다.

---

#### 동작 방식

1. 싱글톤 생성 요청
2. 의존성 리스트 확인
3. 의존 싱글톤 먼저 초기화
4. 현재 싱글톤 초기화 진행

---

### 설계 특징

#### 1. 생명주기 기반 분리
- 객체 유형별 생성 방식 대응
- Unity 구조에 맞는 안정적인 싱글톤 관리

---

#### 2. 초기화 순서 보장
- 의존성 기반 초기화
- 시스템 간 충돌 방지

---

#### 3. 비동기 초기화 지원
- 리소스 로딩과 자연스럽게 연동
- 초기화 지연 문제 해결

---

#### 4. 유연한 선택 구조
- 총 8가지 싱글톤 타입 제공
- 상황에 따라 적절한 싱글톤 선택 가능

---

### 확장성

- 새로운 객체 타입 추가 가능
- 초기화 정책 변경 유연 대응
- 시스템 규모 확장 시 안정성 유지

---

### 확인 포인트

- 객체 타입별 싱글톤 분리 구조
- 동기 / 비동기 초기화 선택 방식
- 의존성 기반 초기화 흐름
