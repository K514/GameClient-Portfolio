## Game Entity

게임 내 모든 오브젝트(플레이어, 몬스터, 탄막, 기믹 등)를  
하나의 공통 구조로 관리하기 위한 엔터티 시스템입니다.

---

### 개요

게임에 등장하는 다양한 객체를 다음과 같이 분류합니다.

- Unit Entity : 플레이어, 몬스터
- Projectile Entity : 탄막, 투사체
- Beam Entity : 레이저
- Gear Entity : 트랩, 환경 기믹

모든 엔터티는 공통된 구조를 기반으로 동작합니다.

---

### 핵심 구조

각 엔터티는 기능을 직접 구현하지 않고,  
**모듈(Module) 단위로 기능을 구성**하도록 설계했습니다.

---

### 공통 모듈 구성

- Physics (물리)
- PathFinding (길찾기)
- AI (상태 머신 기반)
- Action (스킬, 이동, 점프 등)
- Rendering (렌더링)
- Animation (모션)

---

### 설계 의도

기능을 엔터티에 직접 고정하지 않고,  
모듈을 교체하는 방식으로 동작을 변경할 수 있도록 구성했습니다.

---

### 모듈 교체 기반 동작 변경

#### ■ AI 변경
- 적대적인 몬스터 → NPC 전환 가능

#### ■ Action 변경
- 체력 조건에 따라 스킬셋 변경
- 보스 패턴 전환 구현 가능

#### ■ 기능 확장
- 탄막/레이저에 AI 모듈 추가 → 플레이어 추적 가능

---

### 상태 및 전투 개념

모든 엔터티는 공통적으로 다음 개념을 가집니다.

- 체력(Stat)
- 사망(Death)
- 무적 상태

기본적으로:

- 탄막/레이저 → 무적
- 무적 해제 시 → 파괴 가능

---

### 동맹 시스템

각 엔터티는 동맹 정보를 가지고 있으며,  
이를 기반으로 주변 대상의 적/아군을 판별합니다.

---

### 동맹 기반 기능

- 적을 아군으로 전환
- 탄막 방향 전환 후 아군화 (반사)
- 타겟 변경 및 행동 변경

---

### 확장성

- 새로운 기능은 모듈 추가로 확장 가능
- 엔터티 타입 추가 시 기존 구조 변경 최소화
- 다양한 조합으로 새로운 행동 패턴 구성 가능

---

### 확인 포인트

- 모듈 기반 엔터티 구조
- 기능을 “조합”으로 구현한 설계 방식
- AI / Action / Stat / Alliance 분리 구조
- 동적 행동 변경 (런타임 모듈 교체)
