## Pool System

게임 내 객체 재사용을 위한 오브젝트 풀링 시스템입니다.  
생성/삭제 비용을 줄이고, 리소스 수명 관리를 효율적으로 처리하기 위해 설계되었습니다.

---

### 개요

객체를 필요할 때마다 생성/삭제하지 않고,  
미리 생성된 객체를 재사용하는 방식으로 성능을 최적화합니다.

---

### 풀 타입

두 가지 풀을 분리하여 관리합니다.

#### ■ 일반 객체 풀
- 일반 클래스 인스턴스 재사용
- GC 부담 감소

#### ■ Unity 컴포넌트 풀
- GameObject / Component 재사용
- Instantiate / Destroy 비용 최소화

---

### 엔터티 풀 관리 방식

게임 엔터티는 단순 타입이 아닌,  
다음 요소를 조합한 **복합 키**로 관리됩니다.

- 모델 고유값
- 컴포넌트 고유값
- 리소스 수명 타입

👉 동일한 구조의 객체를 정확하게 재사용하기 위한 설계

---

### 리소스 수명 관리

각 객체는 수명 타입을 가지며, 이를 기준으로 관리됩니다.

#### ■ Scene Lifetime
- 씬 단위 객체
- 씬 전환 시 자동으로 풀로 회수

#### ■ (확장 가능)
- 글로벌 객체
- 특정 시스템 단위 객체

---

### 설계 특징

#### 1. 풀 타입 분리
- 일반 객체와 Unity 객체를 분리하여 최적화

---

#### 2. 복합 키 기반 관리
- 객체의 구조를 기준으로 풀을 분리
- 잘못된 객체 재사용 방지

---

#### 3. 자동 회수 시스템
- 씬 전환 시 객체 자동 반환
- 리소스 누수 방지

---

#### 4. 성능 최적화
- 생성/삭제 비용 최소화
- GC 부담 감소

---

### 확장성

- 새로운 객체 타입 추가 시 구조 변경 최소화
- 다양한 수명 정책 적용 가능
- 시스템 단위 풀 관리 확장 가능

---

### 확인 포인트

- 일반 객체 / Unity 객체 풀 분리 구조
- 복합 키 기반 풀 관리 방식
- 리소스 수명 타입에 따른 자동 회수
